# Code of Conduct

## The short version

Be decent. Assume the other person is acting in good faith. Critique the code,
not the person.

## What's expected

- Keep discussion technical and specific. "This breaks on Linux because X" is
  useful; "this is broken" is not.
- Accept that maintainers may decline a change. A rejected PR is a scoping
  decision, not a judgment of you.
- Remember that most contributors are doing this in spare time, in a second
  language, or both.

## What's not acceptable

Harassment, personal attacks, discrimination, or deliberately derailing
discussions. Publishing someone's private information. Sustained disruption
after being asked to stop.

## Scope

This applies in issues, pull requests, commit messages, and any other project
space.

## Enforcement

Report conduct issues privately through
[GitHub Security Advisories](https://github.com/artificemachine/token-diet/security/advisories/new)
(it's the private channel available on this repo) or by contacting the
maintainer directly through their GitHub profile.

Reports are handled privately. Responses range from a warning to a permanent
ban, depending on severity. The maintainer decides; there is no committee.
