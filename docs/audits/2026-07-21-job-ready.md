## Stage 5 — Security / Hardening — PASS (full)

## Stage 6 — Architecture — NEEDS WORK (full) — 2 HIGH
- **HIGH: config-path drift.** `hosts.sh` centralized only `slug|label`. The drift-prone data — each host's config-file path and the `mcpServers`/`mcp`/`servers` key dialect — is still hand-duplicated across ~8 files, and the dashboard carries its own independent Python path list that cannot source the bash registry. This is the remaining Phase 5 work (iters 3-9).
- **HIGH: pinned submodules bypassed on the default install.** `forks/` pins only govern `--local`. The default network path runs `cargo install --git <repo> --force` with no `--rev/--tag`, floating to upstream HEAD; the Serena launcher bakes `uvx --from git+…/serena` so Serena re-resolves HEAD on every invocation. `compat.json`'s "tested" versions are unverifiable against what actually installs.
- **MED (FIXED this session, #57): compat.json dead post-install.** The installed CLI read a nonexistent `~/.local/config/compat.json`, so the version gate was a dead no-op. Now copied by install.sh, removed by uninstall.sh, ICM added, `tested` refreshed.
- LOW: dashboard hard-couples to rtk's JSON schema (no `.get`, no schema-version guard).

Strengths confirmed: `tdconfig` atomicity, marker-guarded idempotency, `TD_MUTATED` mutation tracking + revert hints, globbed (not manifest) lib copy.

## Stage 7 — CI Governance — PASS (full)
Nothing above LOW. Actions SHA-pinned, scanners blocking (`gitleaks` + `Path Leak Guard` required on main), `enforce_admins:true`, `strict:true`, no force-push/deletions, least-privilege `permissions` per workflow, safe idempotent release.yml. LOW: no required PR review (expected for solo).

## Stage 8 — Claims vs Reality — PASS (full)
