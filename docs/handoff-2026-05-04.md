# Session Handoff — 2026-05-04 (v1.7.10 + v1.7.11)

## Executive Summary
Two patch releases that fix MCP registration bugs in the installer. Both shipped, tagged, released, and verified locally.

## v1.7.10 — OpenCode MCP registration

The OpenCode registration block in `scripts/install.sh` was writing to a config that OpenCode does not read. Three things were wrong simultaneously:

| Field | Was | Now |
|---|---|---|
| Config path | `~/.opencode.json` | `~/.config/opencode/opencode.json` (XDG; legacy fallback) |
| Top-level key | `mcpServers` | `mcp` |
| Entry shape | `{command, args}` | `{type: "local", command: [...], enabled: true}` |

Also added:
- **tilth** registration alongside serena
- Windows venv binary detection (`.venv/Scripts/serena-mcp-server.exe`)
- XDG path added to `uninstall.sh` scrub list

3 install.bats tests updated to assert the new shape. PR #48.

## v1.7.11 — Idempotent Claude Code Serena registration

Reinstalls were emitting a false-positive yellow warning:
`Serena MCP: Claude Code setup failed (may already exist)`

Mirrored the Codex CLI pattern: pre-check with `claude mcp get serena`, report `(already configured)` instead. PR #49.

## Stack health diagnostics performed

- Identified that "tilth · ✘ failed" in `/mcp` was a transient session-level disconnect, not a binary problem (handshake works directly).
- Repaired RTK hook missing from `~/.claude/settings.json` via `token-diet repair` (runs `rtk init -g --auto-patch`).
- Recovered `obsidian-semantic` MCP after diagnosing a wedged OrbStack docker engine (socket present, daemon unresponsive). Fix: `orbctl stop && orbctl start`. All 4 containers (mcp-server, dashboard, ollama, postgres) came back healthy.
