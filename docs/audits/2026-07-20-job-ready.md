### 3. Fix the headline claim — you are underselling with a weaker, unsourced number
The README says "40-90%" (`README.md:3`), the GitHub description says "60-90%", `CLAUDE.md:7` says "40–90%". Three figures, no benchmark, no methodology anywhere outside `forks/`. Replace the round unsourced number with the measured one and link the benchmark. An unsubstantiated quantitative claim is the single most damaging thing a senior reviewer can find; here it's also unnecessary.

### 4. Fix the silent config-truncation path (real data-loss bug)
`scripts/install.sh:1456-1467` — verified firsthand:
```python
try:
    with open(cfg) as f: data = json.load(f)
    ...
    with open(cfg, "w") as f: json.dump(data, f, indent=2)
except Exception: pass
```
`open(cfg,"w")` truncates *before* the dump. A failure mid-write leaves the user's `~/.claude/settings.json` empty, and `except Exception: pass` swallows it with no message and no backup. This loop covers `.claude/settings.json`, both `claude_desktop_config.json` paths, and Cowork. It also silently no-ops on malformed JSON, contradicting 7 sibling blocks that abort loudly.

Systemically: exactly 1 of ~15 config-mutation sites uses `os.replace` (`scripts/token-diet:1900`). Everything else is non-atomic. Nothing tests interrupted or partial writes. For an installer that edits other tools' config files, this is the finding a senior reviewer will weigh hardest.

### 5. Decide what Windows support actually is
46 Pester tests exist (`tests/token-diet.Tests.ps1`, `tests/Uninstall.Tests.ps1`) and run in **no CI workflow at all** — `test.yml` is `ubuntu-latest` only, and the ps1 smoke test skips unless `sys.platform == "win32"`. `CLAUDE.md` lists Pester as part of the "full suite" but neither CI nor the pre-commit hook ever runs it. The PS1 is also not at feature parity: `grep -c "ctxwarn\|docextract" scripts/token-diet.ps1` = 0. Either add a `windows-latest` job or demote Windows to "experimental, untested in CI" in the README. Shipping 46 tests that execute nowhere reads worse than shipping none.
